				<h1 id="user-content-mongodb-backup-s3">mongodb-backup-s3</h1>
<p>This image runs mongodump to backup data using cronjob to an s3 bucket</p>
<p>Build Dockerfile check if run.sh is LF instead of CRLF</p>
<h2 id="user-content-usage">Usage:</h2>
<pre><code class="chroma language-text">docker run -d \
  --env AWS_ACCESS_KEY_ID=awsaccesskeyid \
  --env AWS_SECRET_ACCESS_KEY=awssecretaccesskey \
  --env BUCKET=s3bucket
  --env MONGODB_HOST=mongodb.host \
  --env MONGODB_PORT=27017 \
  --env MONGODB_USER=admin \
  --env MONGODB_PASS=password \
  halvves/mongodb-backup-s3
</code></pre><p>If you link <code>halvves/mongodb-backup-s3</code> to a mongodb container with an alias named mongodb, this image will try to auto load the <code>host</code>, <code>port</code>, <code>user</code>, <code>pass</code> if possible. Like this:</p>
<pre><code class="chroma language-text">docker run -d \
  --env AWS_ACCESS_KEY_ID=myaccesskeyid \
  --env AWS_SECRET_ACCESS_KEY=mysecretaccesskey \
  --env BUCKET=mybucketname \
  --env BACKUP_FOLDER=a/sub/folder/path/ \
  --env INIT_BACKUP=true \
  --link my_mongo_db:mongodb \
  halvves/mongodb-backup-s3
</code></pre><p>Add to a docker-compose.yml to enhance your robotic army:</p>
<p>For automated backups</p>
<pre><code class="chroma language-text">mongodbbackup:
  image: &#39;halvves/mongodb-backup-s3:latest&#39;
  links:
    - mongodb
  environment:
    - AWS_ACCESS_KEY_ID=myaccesskeyid
    - AWS_SECRET_ACCESS_KEY=mysecretaccesskey
    - BUCKET=my-s3-bucket
    - BACKUP_FOLDER=prod/db/
  restart: always
</code></pre><p>Or use <code>INIT_RESTORE</code> with <code>DISABLE_CRON</code> for seeding/restoring/starting a db (great for a fresh instance or a dev machine)</p>
<pre><code class="chroma language-text">mongodbbackup:
  image: &#39;halvves/mongodb-backup-s3:latest&#39;
  links:
    - mongodb
  environment:
    - AWS_ACCESS_KEY_ID=myaccesskeyid
    - AWS_SECRET_ACCESS_KEY=mysecretaccesskey
    - BUCKET=my-s3-bucket
    - BACKUP_FOLDER=prod/db/
    - INIT_RESTORE=true
    - DISABLE_CRON=true
</code></pre><h2 id="user-content-parameters">Parameters</h2>
<p><code>AWS_ACCESS_KEY_ID</code> - your aws access key id (for your s3 bucket)</p>
<p><code>AWS_SECRET_ACCESS_KEY</code>: - your aws secret access key (for your s3 bucket)</p>
<p><code>BUCKET</code>: - your s3 bucket</p>
<p><code>BACKUP_FOLDER</code>: - name of folder or path to put backups (eg <code>myapp/db_backups/</code>). defaults to root of bucket.</p>
<p><code>MONGODB_HOST</code> - the host/ip of your mongodb database</p>
<p><code>MONGODB_PORT</code> - the port number of your mongodb database</p>
<p><code>MONGODB_USER</code> - the username of your mongodb database. If MONGODB_USER is empty while MONGODB_PASS is not, the image will use admin as the default username</p>
<p><code>MONGODB_PASS</code> - the password of your mongodb database</p>
<p><code>MONGODB_DB</code> - the database name to dump. If not specified, it will dump all the databases</p>
<p><code>EXTRA_OPTS</code> - any extra options to pass to mongodump command</p>
<p><code>CRON_TIME</code> - the interval of cron job to run mongodump. <code>0 3 * * *</code> by default, which is every day at 03:00hrs.</p>
<p><code>TZ</code> - timezone. default: <code>US/Eastern</code></p>
<p><code>CRON_TZ</code> - cron timezone. default: <code>US/Eastern</code></p>
<p><code>INIT_BACKUP</code> - if set, create a backup when the container launched</p>
<p><code>INIT_RESTORE</code> - if set, restore from latest when container is launched</p>
<p><code>DISABLE_CRON</code> - if set, it will skip setting up automated backups. good for when you want to use this container to seed a dev environment.</p>
<h2 id="user-content-restore-from-a-backup">Restore from a backup</h2>
<p>To see the list of backups, you can run:</p>
<pre><code class="chroma language-text">docker exec mongodb-backup-s3 /listbackups.sh
</code></pre><p>To restore database from a certain backup, simply run (pass in just the timestamp part of the filename):</p>
<pre><code class="chroma language-text">docker exec mongodb-backup-s3 /restore.sh 20170406T155812
</code></pre><p>To restore latest just:</p>
<pre><code class="chroma language-text">docker exec mongodb-backup-s3 /restore.sh
</code></pre><h2 id="user-content-acknowledgements">Acknowledgements</h2>
